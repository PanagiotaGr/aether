# Evaluation

The evaluation layer turns individual simulation runs into comparable results.

## Core metrics

- success flag
- route length
- number of route states
- minimum estimated clearance
- average proximity score
- tail-average score
- runtime

## Benchmark tables

A useful benchmark table should include:

- scenario name
- seed
- configuration file
- commit hash
- metric columns
- notes about failed runs

## Plot ideas

- route length by scenario
- score distribution by seed
- runtime versus grid size
- route visualization in 3D
